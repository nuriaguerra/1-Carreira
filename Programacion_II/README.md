# Programación II

> Programación · Primeiro curso

---

## De que vai esta materia

Programación II continúa onde deixou Programación I, pero dá un salto cualitativo importante: aquí empezan as **estruturas de datos** e a **programación orientada a obxectos**. É o momento en que o código deixa de ser unha secuencia de instrucións e empeza a organizarse de forma máis sofisticada.

Esta materia é das máis importantes da carreira. O que aprendas aquí úsalo en case todas as materias que veñen despois.

---

## Contidos habituais

- Programación orientada a obxectos: clases, obxectos, herdanza, polimorfismo
- Estruturas de datos lineais: pilas, colas, listas enlazadas
- Árbores: binarias, de busca, equilibradas
- Tablas hash
- Introdución aos algoritmos de busca e ordenación
- Complexidade algorítmica: notación O grande
- Xestión dinámica de memoria
- Boas prácticas: modularidade, legibilidade, documentación

## Consellos

- Implementa cada estrutura de datos desde cero polo menos unha vez, a man. Así é como realmente se aprenden.
- A complexidade algorítmica non é un detalle teórico — é o que distingue un código que escala dun que non.
- A recursividade en árbores parece máxica ao principio. Cando a pilles, non a quererás soltar.
- Se tes dúbidas entre usar unha lista, un array ou un mapa: a resposta case sempre depende de que operacións fas máis. Pensa niso.
